What is GIT?

GIT is one of many control systems. It is designed primarily to manage source code ie. the lines of text that we have been experimenting with. It is essentially used during software development. Programmers use GIT to “coordinate their work” as it can track any changes that are made to the file sets. A complete history of the work performed is available for the users to review. It is also designed with speed and the integrity of data in mind.

Why do we need source code Version Control System?

A Version Control System is necessary because without it errors would occur regularly. They work by creating a list of file changes over time. When an individual modifies a file, a new file is created and stored in the repository. This prevents users from modifying an old version of the file or overriding someone else’s work. A lot of communication would be necessary without the Version Control System.
